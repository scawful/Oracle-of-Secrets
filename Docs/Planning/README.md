- Master plan: `narrative_design_master_plan.md`
- Roadmap: `improvement_roadmap.md`
- Story graph: `Story_Event_Graph.md`
- Lore implementation tracker: `lore_implementation.md`
- Item flow review: `item_flow_balance_check.md`
- Inventories: `chest_inventory.json`, `overworld_item_inventory.json`, `receiveitem_calls.json`

## Plans

All focused, single-topic plans live in `Plans/`.

Common entry points:

- Release definition: `Plans/release_2026_definition.md`
- Progression helpers/spec: `Plans/progression_infrastructure.md`
- Goron Mines minecart design: `Plans/goron_mines_minecart_design.md`

## Status / Handoffs

Session summaries, runtime verification artifacts, and ralph diagnostics live in `Status/`.

## Where To Put New Docs

- A new focused proposal/spec: `Plans/<topic>.md`
- A snapshot/handoff/status report: `Status/<date>_<topic>.md`
- A cross-cutting audit/inventory: top-level `Docs/Planning/<name>.(md|json|csv)`
